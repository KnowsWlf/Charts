# docker-proxy

A Helm chart that deploys a set of [CNCF distribution](https://github.com/distribution/distribution) registries on Kubernetes:

- **Pull-through cache mirrors** â€” one per upstream (Docker Hub, GCR, GHCR, Quay, `registry.k8s.io`, MCR, Elastic, NVCR, â€¦). Each caches and accelerates pulls from its upstream.
- **Hosted private registries** â€” an entry *without* an upstream becomes a normal push/pull registry.

Every registry is a `Deployment` + `Service` + `PersistentVolumeClaim` + (optional) Traefik `IngressRoute`s, all generated by ranging over a single `registries` list. Adding a mirror is one line of YAML.

## TL;DR

```bash
helm repo add knowswlf https://knowswlf.github.io/Charts/
helm install docker-proxy knowswlf/docker-proxy \
  --namespace docker-proxy --create-namespace \
  --set domain=example.com \
  --set traefik.tlsSecretName=example-com-tls
```

## How it works

The image is the standard distribution registry, configured via `/etc/distribution/config.yml`. A registry with a `proxy:` block is a pull-through cache; without it, it's a hosted registry. This chart renders that config per entry from the `registries` list:

| Entry has `upstream` | Result |
|----------------------|--------|
| yes | `proxy.remoteurl` set â†’ **mirror** (read-only cache) |
| no  | no `proxy` block â†’ **hosted** registry (push + pull) |

Each registry is served at `<host|name>.<domain>` through Traefik (web â†’ websecure redirect + TLS), matching the original standalone manifests.

## Routing: Traefik or Ingress

Two mutually exclusive options â€” enable one, not both:

- **Traefik** (default): `traefik.enabled=true` generates per-registry `IngressRoute`s (HTTPâ†’HTTPS redirect + TLS via `traefik.tlsSecretName`).
- **Standard Ingress**: for nginx/HAProxy/etc. clusters, set `traefik.enabled=false` and `ingress.enabled=true`. One `Ingress` is created per registry at `<name>.<domain>`.

```bash
helm install docker-proxy knowswlf/docker-proxy \
  --namespace docker-proxy --create-namespace \
  --set domain=example.com \
  --set traefik.enabled=false \
  --set ingress.enabled=true \
  --set ingress.className=nginx \
  --set ingress.tls.enabled=true \
  --set ingress.annotations."nginx\.ingress\.kubernetes\.io/proxy-body-size"="0"
```

> **nginx note:** the default `proxy-body-size` (1m) rejects image layer pushes/pulls. Set the annotation above (`0` = unlimited) so the registry works.

## Image

Defaults to `dqzboy/registry:latest` (the multi-arch rebuild used by the [Docker-Proxy](https://github.com/dqzboy/Docker-Proxy) project, distribution v3 line). It is **config-compatible with the official image** â€” switch to the upstream GA build any time:

```bash
--set image.repository=registry --set image.tag=3
```

Both read `/etc/distribution/config.yml`, so no other change is needed.

### A `registries` entry

```yaml
registries:
  - name: hub                              # â†’ registry-hub, host hub.<domain>
    upstream: https://registry-1.docker.io # mirror; omit for a hosted registry
    host: hub        # optional subdomain override (defaults to name)
    username: ""     # optional upstream credentials (proxy only)
    password: ""
    size: 20Gi       # optional PVC size override
    ttl: 168h        # optional proxy TTL override
    auth: false      # optional; require basic auth (hosted, needs auth.enabled)
```

## Examples

Mirror a pull through the Docker Hub cache:

```bash
docker pull hub.example.com/library/nginx:latest
```

Push to the hosted private registry:

```bash
docker tag myapp:1.0 registry.example.com/myapp:1.0
docker push registry.example.com/myapp:1.0
```

Enable basic auth on the hosted registry:

```bash
helm upgrade docker-proxy knowswlf/docker-proxy --reuse-values \
  --set auth.enabled=true \
  --set auth.users[0].username=admin \
  --set auth.users[0].password=s3cret
# then: docker login registry.example.com
```

## Notes

- Keep `replicaCount: 1` per registry â€” the filesystem storage driver is not safe for concurrent writers on one volume.
- A public registry should serve TLS; set `traefik.tlsSecretName`.
- Pod template carries a `checksum/config` annotation, so editing a registry's config rolls its Deployment automatically.
